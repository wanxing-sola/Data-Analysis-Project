# 电信客户流失数据分析项目
## 项目介绍
基于阿里云电信客户流失公开数据集，共15万条电信用户业务数据，69维特征字段，完整复现运营商后台用户数据分析全流程。
## 技术栈
Python3.8、Pandas、Matplotlib、MySQL、SQLAlchemy、pymysql
## 目录结构
- data/raw：原始数据集（阿里云公开电信客户流失数据集，文件较大不上传仓库）
- data/processed：清洗后标准化数据
- src：整套业务代码（数据读取、清洗、多维度统计、可视化绘图、MySQL批量入库）
- result：输出统计表格、业务可视化图表
## 功能实现
1. 数据预处理：中位数填充数值缺失值，过滤话费、使用时长异常样本；基于月消费金额衍生用户分层标签；
2. 多维度分析：按消费分层、地理区域统计用户使用时长、客户流失率；
3. 可视化：绘制柱状图、折线图直观展示用户流失规律；
4. 数据入库：SQLAlchemy分块批量写入15万条数据至MySQL，pymysql执行分组SQL验证数据。
## 运行说明
1. 本地提前创建数据库 telecom_db；
2. 修改05_save_to_mysql.py内MySQL账号密码；
3. 按顺序执行src下全部py脚本；
4. 依赖库安装：pip install -r requirements.txt
## 业务结论
1. 高消费用户群体月均使用时长高，但流失率偏高，说明高价套餐性价比不足，可推出高端专属优惠、增值语音 / 流量包挽留；
2. 低消费用户话费少、使用时长低，流失率中等，可推出低价小额流量套餐提升用户粘性；
3. 不同地理区域流失差异明显，偏远区域客户使用时长偏低、流失更高，反映通信覆盖、线下服务存在短板，可针对性优化基站与线下营业厅运营。
注：原始数据集来自阿里云电信客户流失公开数据集，文件较大不上传仓库，本地运行需自行配置 MySQL 账号、数据集路径。
